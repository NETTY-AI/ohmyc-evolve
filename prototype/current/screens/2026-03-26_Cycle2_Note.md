# Cycle 2: 디자인 개선 사이클 

## 목적
- 심미적인 완성도 향상
- 캐릭터 홈(Character Home)과 메인 바텀 내비게이션(Bottom Navigation) 시각적 개선
- Apple HIG, Liquid Glass, 그리고 VI의 결합

## 개선사항 (적용점)

1. **캐릭터 홈 (대시보드)**
   - Apple의 프로 카드(Pro Card) 스타일 도입 (모서리 곡률 `20px` 연속 곡선 대응)
   - 앱이 열렸을 때 무대(배경)는 뒤로 물러나고, 캐릭터의 고유 색상이 화면의 대부분을 차지하게 설계 (`UX.md - 유저가 돌아오면 캐릭터가 반갑게 맞이` 원칙 적용).
   - "온도(Temperature)" 개념 시각화 도입: `alive · 36.5°C` 로 캐릭터의 현재 상태를 카드 위에 표시하여 관계성을 직접적으로 보여줌. 
   - 사용자가 가장 많이 하는 두 가지 행동(Chat, Memory)을 별도의 액션 영역으로 분리시켜, 카드의 이미지는 방해받지 않도록 구성. CTA 버튼은 VI의 메인 Tint Color를 활용하여 입체적인 그림자(`Level 3`)를 적용. 

2. **메인 바텀 내비게이션 (Liquid Glass)**
   - Apple HIG(iOS 26)에서 강조하는 **Liquid Glass** 소재 전면 도입.
   - 불투명 배경이 아닌 반투명(`0.72`)과 고강도 블러(`40px`), 그리고 빛의 굴절을 표현하는 미세한 하얀색 테두리 사용.
   - 배경에 있는 콘텐츠(캐릭터 이미지나 스크롤 되는 리스트)가 은은하게 비치도록 하여, 레이아웃의 깊이감을 확보. 
   - 탭이 활성화 되었을 때, Tint Color가 뚜렷하게 발색되며, 비활성 탭과의 색상 콘트라스트를 명확하게 설정.

## 추가 개선사항 (Discover 뷰 기반)

3. **캐릭터 홈 (Discover 뷰 기준의 그리드 모델 적용)**
   - 최상단 검색창(Search Bar)을 프로필 아이콘 대신 Liquid Glass 패턴으로 변경하여 통일성 부여.
   - 상단 장르 칩(Pill Chips) 스크롤 영역 추가. 활성 칩(`active`)은 Tint Color(`--tint`)를 사용해 직관적인 피드백 제공.
   - 히어로 카드(최상단 풀사이즈 카드)와 2열 그리드 카드를 분리 배치. 히어로 뷰는 가로 너비를 꽉 채워 주목도를 극대화.
   - 오버레이 그래디언트의 가독성을 높이기 위해 상하 비율 및 불투명도를 조정. (`linear-gradient(to top, rgba(0,0,0,0.85) 0%, rgba(0,0,0,0.4) 60%, transparent 100%)`)
   - `Breathe` 모션 애니메이션의 이징 커브를 Apple의 `cubic-bezier(.22, 1, .36, 1)`에 가깝게 조정하여, 자연스러운 심장 박동처럼 보이도록 세부 튜닝.
   - 각 카드별 Breathe 모션의 `animation-delay`를 엇갈리게 설정하여, 리스트 전체가 동시에 깜빡이는 기계적인 느낌 탈피.

4. **바텀 내비게이션 비율 변경**
   - 텍스트 라벨을 삭제하고 아이콘(24px) 중심으로 과감하게 재편. 활성화 상태는 후면에 Tint 배경 페이드인 투명 효과(Pill Shape)를 주어 시각적 강조.
   - Create(+) 탭은 중앙에 별도의 볼륨감(Shadow + Tint background)을 주어 액션의 중심(Hub)임을 명확히 함.
   - 전체를 앱 하단 모서리 থেকে 띄워 올린 플로팅(Floating) Glass Bar 형태로 제작, Scroll 시 동적 크기 변환(Shrink)이 적용 가능한 기초 뼈대 구축.

## 디테일 수정 내역 (2차 피드백 반영)

1. **`Home_Tab` 탭바 통합**
   - 기존의 하단 고정형, 4탭 구조를 폐기하고 `Discover` 화면에서 채택된 "플로팅 Glass Bar, 라벨 제거, 5탭, Create(+) 차별화" 디자인으로 통합.
   - 이제 두 화면이 완전히 동일한 탭 내비게이션 아키텍처를 공유함.

2. **Breathe 이징(Easing) 동기화**
   - `Home_Tab` 화면의 Breathe 모션 CSS 애니메이션을 `ease-in-out`에서 `cubic-bezier(.22, 1, .36, 1)`(Apple Smooth Easing)으로 변경. 
   - 전역 CSS 변수 `--ease-smooth`를 도입하여 모든 애니메이션(칩, 탭, 스크롤 등 향후 확장 포함)의 동적 움직임 통일.

3. **히어로 카드(Hero Card) 규격 2:3 복귀**
   - Discover 시안의 4:5 비율을 폐기하고, VI 가이드의 원칙에 따라 `2:3` 종횡비와 모서리 곡률 `20px`로 복원시킴.
   - 단일 원칙(무대 위 모든 카드는 2:3, 단일 캐릭터 아트 중심)을 유지하되, 히어로 섹션은 화면 가로 영역을 모두 차지하는 `width: 100%` 구조로 크기감만 키워서 주목도를 유지함.
   - 비율이 세로로 길어졌기 때문에 타이포그래피 공간이 충분히 확보되어, 오버레이 가독성 이슈도 완화됨.
